Understanding the Data

The data consists of a series of variables collected by Devvy that track bike usage in the North-East region of the US. Devvy is a company that offers subscription-based bike leases to customers, with there being two different types of bike subscriptions: annual subscriptions for "members" and daily subscriptions for "casual" customers. The bikes can be unlocked from one station and returned to any other station in the system anytime. Bike usage is very important for the brand as it helps understand consumer behavior and adjust marketing and business strategies accordingly.

The data has been downloaded and pre-loaded into a SQL database. A detailed description of all variables contained in the dataset is presented below.

# Variable Database variable Description
1 ID ride_id This variable serves the purpose of uniquelly identifying a single trip in the dataset.
2 Rideable Type rideable_type Shows the type of bycicle used in a particular trip.
3 Start time started_at Tracks the time at which a trip took place initially.
4 End time ended_at Tracks the time at which a trip finished.
5 Start Station start_station_name Name of the station at which the trip started.
6 Start Station ID start_station_id Unique ID of the station at which the trip started.
7 End Station end_station_name Name of the station at which the trip ended.
8 End Station ID end_station_id Unique ID of the station at which the trip ended.
9 Start Latitude start_lat Initial latitude of the trip, that is, the latitude of the start station.
10 Start Longitude start_lng Initial longitude of the trip, that is, the longitude of the start station.
11 End latitude end_lat Final latitude of the trip, that is, the latitude of the end station.
12 End Longitude end_lng Final longitude of the trip, that is, the longitude of the end station.

How is the data being processed?

  1. Through a series of variable-selection fields you will be prompted to select which variables you want to analyze.
  2. Once you select the variables, a SQL query will be run on the database to filter data relevant to the query.
  3. Once the data has been selected, a series of processes will be carried out to prepare the data and plot it.

Note that not all variables might be available for you to query over.
